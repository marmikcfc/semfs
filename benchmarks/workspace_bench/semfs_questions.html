What question is each command really asking — and how a semantic workspace answers it

Every file command is an agent asking a question. POSIX answers the literal question (by name, exact text, path, bytes). But the question the agent actually has is semantic (by meaning, relevance, relationship). On a plain filesystem the agent must decompose its real question into many literal ones — and guess. A semantic workspace lets it ask the real question directly.

The shift: the unit stops being a command and becomes a question. POSIX commands answer literal questions about names, bytes, and metadata; the agent's real questions are about meaning, relevance, and relationships. The semantic workspace collapses "decompose-into-many-literal-questions-and-guess" into "ask the real question once."
