The line the executions draw:
  • Discovery under uncertainty (find, grep, exploratory ls) → semantic alt YES. This is where the agent is guessing where/which; literal POSIX forces enumeration and burns tokens. Every wasted call in all four runs clusters here.
  • Deterministic ops (cp mv rm mkdir write, exact cat <known file>) → keep literal. The agent knows the exact target; fuzzing them adds risk and can drop data. Only the index side-effect is automatic.
  • The format trap (file+openpyxl+pandas+HTMLParser, 3 of 4 runs) → not a search problem. Solved by transcription (pre-normalized text) so a literal cat works — not by a "semantic cat".
