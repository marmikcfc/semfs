`ls` → Knowledge-Graph: semantic readdir

Make the agent's reflexive first move return relevance, not a file dump · case 289 · 2026-06-06 · parent: case289-retrieval-investigation

The lever, in one line. Token cost is dominated by the agent's first move. With an identical full-P0 config, when codex greps-first it costs 35,241 tokens / 4 calls (≈ cloud's 26,598 / 3); when it os.walk-first it costs 92,591 / 7. The agent enumerates first by default and you cannot instruct that away — so make the orientation move itself semantic.

The original idea, judged

🟢 Steelman
  • Rides the move you can't suppress — the agent always enumerates first.
  • No hint compliance: every agent runs ls reflexively; meets it where it looks.
  • Exposes in-language entities (成交金额,转化率) → correct query formation → fixes cross-lingual.
  • Truest expression of a "semantic filesystem."
🔴 Strawman
  • Raw readdir is load-bearing POSIX: cp -r, globbing, and writing to model_output/ need REAL entries.
  • A graph doesn't map onto per-directory readdir.
  • Agents parse ls as filenames → graph-text confuses them (cf. grep-header +53%).
  • Needs a rich KG (today 21/595 files) + caching on a hot syscall.
Verdict: brilliant kernel (ride the reflexive move), wrong vehicle (don't break readdir). Keep readdir POSIX-clean; deliver the KG through a channel the agent already uses → the 3 approaches below.
