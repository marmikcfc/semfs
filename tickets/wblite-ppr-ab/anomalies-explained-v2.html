② Validating the original hypothesis ("plain is maxed exactly there")

CONFIRMED as a correlation — exactly.
Cross-persona rankings: kaifa = lowest plain tokens (381K < 472 < 619 < 723) and houqin = highest plain accuracy (27.9% > 19.7 > 17.5 > 9.8). The anomaly lands on the exact persona where plain peaks on that axis. The original call deserves credit: the pattern is real and precisely stated.

houqin accuracy drop — fully explained by the hypothesis. Decomposing the drop: plain 27.9% (HIGHEST plain) vs PPR 21.8% (rank 3/4 — mid, not catastrophic). So the gap is driven by plain being exceptional, not by PPR collapsing. "Plain is maxed" holds. ✓

③ Where the hypothesis is incomplete (kaifa)

The kaifa token rise is only HALF "cheap plain."
kaifa plain = 381K (lowest, ✓) — but kaifa PPR = 619K is also the most expensive PPR of any persona (rank 1/4). So the rise is cheap-plain AND expensive-PPR. The ceiling story explains the cheap baseline but is silent on why kaifa's PPR is uniquely costly.
And ~half of it is a mean artifact.
Token cost is fat-tailed (medians ~250–310K everywhere; outliers 2–4M). kaifa PPR mean 509K but median 275K (mean = 1.9× median). The anomaly table is mean-based, so the +237K overstates the typical-cell rise — much of it is the same outlier tail (case 226 etc.), not a uniform PPR tax.

⑤ Verdict (forming-opinions)

Original hypothesis ("one phenomenon: plain is maxed"): ~60% — the correlation is exact and houqin is well-explained, but it (a) only half-explains kaifa (omits the uniquely-expensive PPR), (b) is mean-based (the rise shrinks at the median), and (c) rests on n=4 personas — a 4-point correlation is weak as a law.

Refinement — workspace structure is the common cause: ~70%. Code-vs-docs structure drives both the plain-peak (A's observation) and the PPR-failure mode (the half A misses): dense code graph → over-retrieval → tokens; browsable doc tree → semantic distractors → accuracy. "Plain is maxed" is the symptom; structure is the root.

Strongest case against (steel-manned): with only 4 personas, both stories are post-hoc fits; the true driver could be per-persona task type (kaifa has more enumeration tasks → crawl/over-read; houqin has more navigational tasks → plain wins) independent of code-vs-docs.

I'd change my mind if: (1) kaifa PPR does not read more distinct files than other personas → kills the "dense graph → over-retrieval" half of B; (2) houqin's failed PPR cases do not show reads of semantically-near-but-wrong docs → kills the "distractor misdirection" half; (3) on MEDIAN tokens (not mean), kaifa's rise vanishes → the token anomaly was C (artifact) all along.

⑥ Caveats

  • n=4 personas — every "law" here is a 4-point fit. Treat as a hypothesis to test on more personas, not a result.
  • Token anomaly is mean-based — fat-tailed distribution; the median tells a calmer story.
