01 What a "turn" costs — the loop that generates the bill

The agent (codex/GPT-5.4) runs a loop. With cached_input = 0 on this endpoint, the model re-reads everything accumulated so far on every iteration — and we are billed for every re-read.

the agent loop, honestly pricedcost model
def agent_session(task):
    context = [SCAFFOLD, task]            # S ≈ 3K tokens — re-sent EVERY turn
    bill = 0
    while not done:
        bill += tokens(context)           # ← the whole history, re-paid, every turn
        action = llm(context)             # model decides the next tool call
        out    = run_tool(action)         # find / cat / semfs grep / write ...
        context += [action, clip(out)]    # appended FOREVER (clip() is §03)
    return bill

# Consequence (Law II): a tool output of size o, emitted at turn s,
# is re-paid on every later turn:
#     cost(o) = o × (T − s)        ← early big outputs are the most expensive
So "deleting a turn" doesn't just save that turn's action — it deletes one full re-read of the scaffold plus every byte of history. Turn count is the multiplier on everything else.

03 The harness clip — the layer nobody tells you about

Between every tool and the model sits codex's truncation layer. The semfs render is not what the model reads — clip(out) is. Reverse-engineered from this morning's probes (codex 0.133.0):

inside codex, between tool and model (reconstructed)measured behavior
def clip(tool_output):
    if size(tool_output) <= SAFE_BUDGET:          # measured: ≤ ~10 KB passes WHOLE
        return tool_output                        # (9.8 KB / 200 lines: untouched)
                                                      # (~15 KB: boundary notice appears)
    return ( head(tool_output, ≈600 tokens)       # overflow is CATASTROPHIC:
           + f"…{n} tokens truncated…"              #   49 KB in → ~1.2K tokens out
           + tail(tool_output, ≈600 tokens) )      #   the MIDDLE is silently gone

# NOT a 256-line cap on this build — a 300-line/5.4KB file passed whole.
# The notice is token-denominated ("…11050 tokens truncated…").

04 E6 — how we measured the clip without reading codex's source

The trick: make the model itself report which bytes survived. Feed it a file of numbered marker lines, ask it to read the file with one command and report the first marker it saw, the last, and the missing range. The gap in the numbers IS the clip, measured from the inside.

the E6 probe (ran on the box this morning, /tmp/e6/)experiment
def probe(n_lines, line_bytes):
    write_file("markers.txt",
               [f"MARKER-{i:04d} " + "x"*(line_bytes-13) for i in range(1, n_lines+1)])
    return codex_exec("cat markers.txt — then report ONE line: "
                      "FIRST=<n> LAST=<n> MISSING=<range|NONE> TRUNC=<notice|NONE>")

probe(300,  18)   #  5.4 KB → FIRST=1 LAST=300  MISSING=NONE      ⇒ no 256-line cap
probe(200,  49)   #  9.8 KB → all 200 visible                     ⇒ ≤10 KB is safe
probe(330,  47)   # 15.5 KB → boundary truncation notice          ⇒ the cliff starts
probe(1000, 49)   # 49 KB  → FIRST=1 LAST=1000 MISSING=50–951     ⇒ head+tail window,
                  #          TRUNC="…11050 tokens truncated…"       ~1.2K tokens survive
Why this matters for w1: the scout stack's grep render came out at 13.2 KB — inside the safe window. The model saw every ranked hit and every answer row. Under the OLD uncapped render (~300 KB), clip() would have gutted it to ~1.2K tokens of head+tail, the agent would have lost confidence, and the crawl spiral would have started — which is precisely the 247–700K-token disasters in the original matrix.

05 How semfs keeps the payload inside the window

The fix shipped on 2026-06-11 in crates/semfs/src/cmd/grep.rs: a per-hit render cap with an honest marker, knob SEMFS_GREP_RESULT_CAP.

grep.rs render path (pseudocode of the actual patch)shipped
def render_grep_results(hits,
                        RESULT_LIMIT    = 5,      # env SEMFS_RESULT_LIMIT
                        GREP_RESULT_CAP = 6144):  # env SEMFS_GREP_RESULT_CAP (bytes/hit)
    out = []
    for hit in hits[:RESULT_LIMIT]:               # ranked by vector+FTS fusion + rerank
        text = hit.extracted_text                  # for xlsx: the readable table,
                                                   #   not binary bytes (format-trap fix)
        if len(text) > GREP_RESULT_CAP:
            text = text[:GREP_RESULT_CAP] + "\n[TRUNCATED — open the file for the rest]"
        out.append(f"#{hit.rank} {hit.path}\n{text}")
    return "\n".join(out)
# w1 actual: 5 hits → 13,244 bytes total → passed clip() whole.
Known sharp edge (found writing this page): the guarantee is per-hit, not global. Worst case is RESULT_LIMIT × CAP = 5 × 6 KB = 30 KB — past the ~15 KB cliff. w1 passed because real hits are smaller than the cap. The robust fix is a global render budget (cap_total ≈ 10 KB, answer-bearing top hit funded first) — logged as a refinement for E9's two-tier arm.

07 How we save turn cost — the playbook as code

Updated after the full evening: 6 scout-class runs + 3 plain bars. Each avoided turn is worth (scaffold + accumulated history) — on this task ≈10–15K tokens per late turn. Five mechanisms are shipped (commit eae0980); the sixth is the open frontier.

turn_savings.py — every mechanism, its status, its measured worthshipped + frontier
# 1. NEVER COMMAND A READ                      [shipped · deterministic]
hint.remove("read kg/KNOWLEDGE_GRAPH.md FIRST")
#    worth: ~58K/run (35.7K turn-1 read + 22.4K crawl cascade)
#    result: 0 KG reads, 0 full-tree crawls in all 6 scout-class runs

# 2. MAKE TURN 1 TERMINAL-GRADE                [shipped · probabilistic]
render = rank(hits)[:5]                       # answer rows inline,
assert size(render) < CLIP_WINDOW             # nothing silently gutted
#    when codex trusts it: search → write = 2 calls, 21K tokens (hit 2/6 runs)

# 3. ESCAPE HATCH OPEN BUT UNNECESSARY         [shipped · deterministic]
SEMFS_SEARCH_ONLY = "off"                     # the real tree stays readable
#    kills the spiral CLASS: zero 45-min timeouts / 23-call flails since

# 4. CLOSE DEAD-ENDS EARLY (provenance check)  [shipped · ~2/3 compliance]
for f in task.named_sources: peek(f, 400)    # see the 403 ONCE → stop hunting
#    w2 burned 6 grep variants hunting data that does not exist;
#    wp1 peeked, reported, scored 6/15 (+2 rubrics) at 94K

# 5. PAY BYTES ONCE, HONESTLY                  [shipped · deterministic]
if capped: render += "# ^ TRUNCATED — open the file for the rest"
#    a silently-gutted blob breeds re-query turns; an honest marker
#    converts them into ONE targeted read

# 6. THE FRONTIER: STOP SIGNAL IN THE RENDER   [E9 · next]
render += f"# confidence: {agreement(vector,fts,rerank)} — " \
          f"{'no further search needed' if high else 'verify by reading'}"
#    the residual 2-vs-9-call coin flip is codex's verify instinct;
#    a static hint can't pin it — per-query confidence might
Net effect, measured (case 289): plain 71–322K tokens / 7–15 calls (mean 171K) → scout-class 21–169K / 2–12 calls (mean 82K, −52%), with W′ accuracy at plain's clean mean (6/15). The deterministic mechanisms moved the whole distribution down; the remaining spread is the agent's own policy — mechanism #6's job.
