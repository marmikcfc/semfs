The agent decides

codex, turn 1

At session start codex auto-read AGENTS.md from its working directory (its native instruction-file mechanism — we never touch the prompt). The hint's economy says: one semantic search first. The model emits a tool call:

/bin/bash -lc 'semfs grep "best selling product" .'
20 chars of query
what used to go wrong hereHint v1 commanded "read kg/KNOWLEDGE_GRAPH.md FIRST" → a 35.7K-token read at the worst possible turn, re-paid on every later turn (~58K/run damage). Deleted in hint v2.

CLI wakes up, finds its container

crates/semfs/src/cmd/grep.rs

The semfs binary resolves which mounted container "." belongs to via the mount marker (tag chanpin-matrix), and reads its env knobs:

SEMFS_RESULT_LIMIT=5SEMFS_GREP_RESULT_CAP=6144SEMFS_GREP_TOTAL_CAP=10240SEMFS_GREP_RENDER_MODE=two-tierSEMFS_REWRITE=1

Query rewrite — the cross-lingual bridge

LLM call (OpenRouter) · skipped if no key

An English query against a Chinese corpus would miss on keywords. One small LLM call expands it (real output from the trace):

# rewritten query: "best-selling product, top-selling product, popular product, 畅销产品, 热销商品, 热卖产品"
~90 chars
why this existsOn the cross-lingual investigation this single step moved the answer file from rank #417 to rank #1 — the one thing literal grep can never do.

IPC to the mount daemon

daemon/protocol.rs · Request::Search over unix socket

The CLI doesn't open the index itself — it asks the daemon that owns the mount. That's what makes the search backend-agnostic: the daemon answers identically whether the index is SQLite+vec, embedded pglite, external pgvector, or cloud. (If the daemon is unreachable: SQLite falls back to a direct read-only open; pglite fails closed — its index lives only in the daemon.)

in: query + scope pathout: Vec<SearchHit> {filepath, chunk, memory, similarity}

Embed the query

gemma-q4 ONNX, local, no network

The rewritten query becomes a 768-dimensional vector — the same space every chunk was embedded into at import.

768 floats ≈ 3 KB
old infra bug hereThe embedder used to cache 898 MB of model blobs INTO the corpus directory → a later unscoped import re-indexed its own model files → mount hang. Fixed by the driver's cache strip (RC4).

Retrieve twice

sqlite-vec KNN + FTS5

Two independent searches over the chunks: vector KNN (meaning) and FTS keyword (exact terms — the rewrite's 热销商品 helps here). Two ranked candidate lists.

vec: best_selling_product_core_data_list.txt (chunk 1, 2) apparel_product_shooting_sheet.xlsx … fts: best_selling_product_core_data_list.txt (畅销/热销 hits) top10_product_status_table.xlsx [semfs: error-page annotation] …

Fuse — RRF

reciprocal-rank fusion

The two lists merge by reciprocal rank: a file ranked well in both lists wins. Robust — but it compresses the score range: fused scores end up packed close together.

live bug found by E9The confidence verdict (station 9) computed its margin on these RRF-squashed scores → the top-vs-#2 gap never cleared 15% → it said MIXED in every run. The HIGH stop-signal has never actually fired. Wave-2 fix: normalize by the result-set spread, (s1−s2)/(s1−sN).

Rerank + cut

JINA cross-encoder (CPU ~45 s on the box) · RESULT_LIMIT

A cross-encoder reads query + each candidate together and rescores — slower but sharper than vector distance. Then the list is cut to RESULT_LIMIT=5. The answer file holds rank #1.

5 hits survive

RENDER — the E9 battleground

grep.rs render loop · cap_render / render_mode / confidence_line

The hits become text for the agent. Rules that apply in every mode: binary hits are served from extracted text, never raw bytes (the format-trap fix); a hit whose excerpt is the whole file gets # ^ COMPLETE FILE; a capped one gets an honest # ^ TRUNCATED; the 403 stub's [semfs: error-page] annotation prints verbatim; and a global 10 KB budget stops the total from ever reaching the harness clip cliff.

The clip — codex's secret chopper

inside the codex harness; measured by E6

Between the tool and the model sits codex's truncation layer (measured this morning, codex 0.133): ≤10 KB passes whole · ~15 KB gets a notice · 49 KB collapses to ~1.2K tokens of head+tail, middle silently gone.

why station 9's caps existAn uncapped broad grep used to emit ~300 KB. The chopper gutted it blind; the agent received shredded results, lost trust, and re-searched — the 247–700K-token death spirals of the original matrix started right here.
