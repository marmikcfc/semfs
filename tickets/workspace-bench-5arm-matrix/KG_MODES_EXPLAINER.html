Experiment Advice

Arm What it really tests Signal quality
plain Raw crawl / grep / cat workflow with no SemFS retrieval help. Required baseline.
oc + dedup + prompt + compress The current best semfs delivery stack from the evo run. Good current ship candidate.
oc + dedup + prompt + compress + current KG SemFS plus agent-visible KG artifacts and current L7/co-mention behavior. Mostly re-tests a known risk: overlay-driven exploration and contamination.
oc + dedup + prompt + compress + ideal hidden KG SemFS plus internal KG routing/priors, with no `/kg` browsing surface. High-value experiment, but only after that mode actually exists.

Bottom line: plain vs current best vs current best + surfaced KG is not the experiment I would spend money on. It mostly tells you whether the known-bad overlay still hurts. The real experiment is plain vs current best vs current best + hidden-internal KG only, after you separate `KG_INTERNAL` from `KG_SURFACE`.

What I Would Test Next

If you want an opinion-forming experiment before building the full hidden-KG mode, the useful control split is:

1. plain 2. current best semfs config 3. current best semfs config + current KG surface Expected outcome: - arm 2 should be the serious contender - arm 3 probably underperforms because it mixes retrieval with graph browsing overhead What this does NOT tell you: - whether hidden KG routing would help - whether KG as an internal prior can cut tokens
The good experiment is not “KG on vs KG off” in the current product shape. It is “hidden KG prior on vs hidden KG prior off” with the agent-facing graph surface held off in both arms.
