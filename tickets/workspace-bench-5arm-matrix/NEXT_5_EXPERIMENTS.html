Next 5 experiments — cut tokens, hold accuracy

2026-06-12 · synthesized from RUN_MANIFEST.md + E8/E11 Modal run-2 artifacts + EC2 trace breakdowns · method: first-principles ▸ theory-of-constraints ▸ five-whys-plus ▸ analyze-benchmark-results (nested combination)

ELI5 → Expert in 5 steps

  1. ELI5: The agent pays for its whole conversation again on every tool call. Long conversations are what's expensive — not big search results anymore.
  2. Why: cached_input=0 ⇒ Tokens ≈ T·S + Σ(T−s)·oₛ + G. Codex also clips every tool output at ~10 KB before the model sees it (E6), so per-call payload oₛ is already bounded. The only unbounded term left is T, the turn count.
  3. Evidence: the 30-cell E8 scatter below — tokens track calls superlinearly; the cell means are dominated by right-tail runs (18–27 calls → 400–900K). Score is nearly independent of tokens within an arm (289 nokg: 113K→0/15, 659K→9/15; E9w2: 49K and 238K both 4/15).
  4. Where tails come from (trace-verified): the 17-call v4_i2 run contains only 40K chars of total tool output — no blob. Its bill is 4 consecutive near-empty semfs grep retries (~400 chars each) + ~8 python rglob crawls, each re-paying 8–10K of context. Empty results breed turns; crawls breed turns.
  5. Expert framing: post-grep-cap, the constraint (ToC) moved from payload to turn-tail. Every proposed experiment must collapse T's right tail — on the hit path (search answers in 1 turn, trusted), the miss path (a miss is terminal, not an invitation to retry), or the discovery path (the map is pre-paid in S, not re-derived imperatively).

Counter-analysis / what else was considered
  • Infra? Run-2 log has zero malformed/falling back to cloud markers; the .semfs-marker and provider fixes were applied; smoke passed. Not infra this time.
  • Payload blobs? Killed by grep-cap + codex clip; v4_i2's total tool output was 40K chars in a 158K-token run. The blob era is over.
  • Variance only? Tails recur in every cell ever measured (5↔16, 2↔12, 5–6↔14, 8–10↔27). Bimodality is structural, but n≥5 + medians are still mandatory.

The 5 experiments (priority order)

1 · E16 — Terminal-miss + repeat-query guard NEWaccuracy-proof

What: render-side miss semantics. (a) When a query returns 0 hits or only sub-floor scores: one authoritative line — “No indexed content matches; nearest terms: …; best next action: read <top dir>” — instead of a thin noise render. (b) Daemon session memory: if a query embeds ≥0.9 similar to a previous one, return a ~50-token “identical results to query #N” stub.

Why: attacks RC1, the literal trace pattern (4 consecutive re-greps). Case 15 is the perfect arena: its task references a missing file, both arms burn 78–894K hunting, and accuracy is provably frozen at 1/16 in all 6 cells — any token saved is free.

Prediction: tail-mode frequency halves on 15/95; case-15 nokg mean −40%+. Kill: tails persist with miss-stubs in trace → tails are crawl-driven, weight shifts to E13. Cost: ~1 day (render + daemon session table).

2 · E14 — semfs slice / search-session artifact REMAININGTHE 80% ONE

What: semfs grep stores the full ranked result server-side, returns ≤1KB (top-1 answer-bearing excerpt + paths + result-id); semfs slice <id> --file N --range a:b | --grep term fetches exact verbatim rows on demand.

Why this is the 80% lever: the floor flow search → slice → write is T=3 ≈ 21–30K tokens — and it has already happened naturally twice (21,473 / 21,743, incl. the 2-call w1 run measured in this analysis at 20.7K). Against E8's plain-289 bar of ~107K that is −75–80%; against current nokg-289 mean (390K) it is −92%. E14's job is to make the floor run the modal outcome instead of a 1-in-3 coin flip: the slice affordance is what makes a small first payload safe to trust (the agent knows it can fetch exact rows without re-searching or re-reading whole files). Field priors: Firetiger artifacts 94% one-shot; Anthropic code-exec −98.7%; SWE-grep context isolation.

Accuracy guard: slice returns verbatim rows (dual-store lesson: tables never compressed); PwC's file-based collapse doesn't apply — the excerpt still carries the answer inline. Kill: agent ignores slice in ≥2/3 runs (affordance unused → hint/UX problem, not architecture). Cost: the one real code investment; scope ~2–3 days.

3 · E9-final — render-mode duel settled properly: paths vs two-tier, all 5 cases, n≥5, median-judged REMAINING

What: the E9 duel run to completion with the discipline the campaign now mandates (n≥5, medians + tail-rate reported separately), plus a third arm on 289 only: per-hit factual metadata line (size, detected content-kind e.g. “HTML error page, 321B”).

Why: paths mode is the most consistent arm ever measured (85.5K ±1K, 7–8 calls, ×2) — consistency IS tail-killing; but it crawled on the multi-file synthesis case (95: 32 calls/424K). The duel decides the per-case-shape default. The metadata line moves 289's 403-discovery from a 20-turn hunt to turn 1 — token win that can only help the 403 rubrics (RC5), and it's delivery design (the defensible form per the integrity-banner line), not coaching.

Prediction: paths wins QA-shaped (289/175) on median + tail-rate; two-tier wins synthesis (95/15). Kill: no arm separates on medians at n=5 → render mode is exhausted as a lever. Cost: runs only, no code (metadata line ~2h).

4 · E13 — ≤1KB workspace map in the hint REMAININGaccuracy-proof

What: top-level dirs + file-type census + “where things live” one-liners injected at mount (S grows ~250 tokens; pre-paid once per turn instead of re-derived imperatively).

Why: attacks RC2 — v4_i2 spent ~8 turns on python crawls locating directories the index already knows. Pure S-for-T trade: +1KB scaffold to delete 2–6 discovery turns. Retrieval-free, so accuracy can't move. Aider repo-map / hermesagent lazy-bootstrap precedent.

Prediction: discovery probes 4–8 → 1–2; −15–30% median tokens on crawl-heavy cases (95/15). Kill: probe count unchanged → maps don't steer codex; drop. Cost: half-day.

5 · E11-v2 — discovery corpus that actually stresses discovery REDESIGNaccuracy-proof

What: rebuild the E11 corpus so linear scanning is genuinely expensive: ≥1,000 files and/or ≥10KB per file (padded realistic prose), unnamed answer files, many near-duplicate siblings, tasks stay cross-lingual (zh task ↔ en files).

Why: E11-v1's null result is a corpus artifact, not a thesis failure — 200×1KB files cost only ~70K to scan, so there was nothing for retrieval to save. The token delta vs plain scales with scan cost (plain grows linearly with corpus; semfs stays flat); this is the arena where the −97% discovery-phase numbers (GrepAI) live, and where the already-proven #417→#1 cross-lingual rewrite becomes decisive. semfs already won 1/2 cells even on the trivial corpus (−28% tokens).

Prediction: plain tokens ≥2× semfs at equal accuracy on ≥2/3 cases. Kill: semfs ≤ plain here → the problem is deeper than arena choice (O8's falsifier). Cost: 1 day authoring (fix the rubric name-strictness — accept “Widget-Sigma (variant 42)” — while in there).
