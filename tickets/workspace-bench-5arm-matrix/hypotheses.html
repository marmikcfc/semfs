Level 1 · Explain it like I'm five

An intern, a librarian, and a very strange pay rule

Forget software for a minute. Here is the whole case as a story about reading.

The intern (the agent, codex/GPT-5.4)

Works alone in a room of 1,452 documents

We give an AI intern a task like "find this quarter's depreciation numbers and write a report." The intern can open files, search, and write — but every action costs money, measured in tokens (roughly: pieces of words read or written).

The strange pay rule (no prompt cache)

Before every action, the intern re-reads its entire notebook

This intern has no memory between actions. Every single time it does anything, it first re-reads everything it has seen so far — and we pay for every re-read. So one careless early decision (opening a huge useless file first) gets paid for again and again, on every later step.

The librarian (semfs)

Our product: a smart filing system

semfs mounts the documents through a smart layer: it can search by meaning (not just exact words), it converts unreadable spreadsheets into readable text, and it leaves the intern a note (AGENTS.md) explaining how to use it.

The question on trial

Does hiring the librarian save money?

Hypothesis: with the librarian, the intern reads less junk, finds things faster, and the bill goes down — without the work getting worse. The first trial said no — the intern did better alone. This case file is about why that verdict was wrong, what's actually true, and what we test next.

The one sentence to remember

The bill is not decided by how smart the search is. It's decided by how the intern behaves — how many actions it takes, how big each thing it reads is, and how early it reads the big things.

Level 4 · First principles

The physics of where tokens go

Strip away every convention and ask what the bill is made of. With no prompt cache, every action re-pays the whole accumulated context. That single fact generates three laws — and one hidden mechanism nobody had noticed.

TotalTokens ≈ T·S + Σₛ (T−s)·oₛ + G
T = number of actions (turns) · S = scaffold re-read every turn (prompt + hint)
oₛ = what action s put into context · (T−s) = how many times it gets re-paid · G = generated output
I.

Turn count multiplies everything. Every extra action re-pays the scaffold and all history. The 5-call run cost 76.8K; the same config making 16 calls cost 97.8K.

II.

Early big reads are the most expensive reads. A read at turn s is re-paid (T−s) times. The hint-commanded 35.7K KG read happens at turn 1 — the single worst position in the whole run.

III.

Even the winner is wasteful. Decomposing plain's 79K on case 289: its first two find probes returned ~40K characters that get re-paid on every later turn — ~60–70% of plain's bill is plain re-reading its own discovery probes. The theoretical floor (1 search → 1 read → 1 write) is ~17–22K. Everyone is ≥3.5× above it.

Exhibit A · A hidden mechanism, discovered 2026-06-11

While verifying the equation against real traces, the numbers refused to add up. The winning cloud run on case 95 emitted a staggering amount of tool output — yet billed a fraction of what the equation predicts:

830,726

→

Conclusion: the codex harness silently truncates tool output before the model ever sees it. There is an uncontrolled clipping layer between semfs grep and the agent — and it clips blindly, possibly cutting the ranked list or the answer row. This reframes the grep-cap fix entirely: its value isn't "fewer bytes," it's choosing which bytes survive instead of letting the harness choose at random. The clip's threshold and shape are unmeasured — which is why experiment E6 runs before everything else.

Level 5 · Expert

Eight opinions, held at gunpoint

Each conviction below carries an explicit credence (how sure we are, as a number) and a falsifier (the concrete observation that would kill it). An opinion without a falsifier isn't an opinion — it's an identity. Click each one to see the full case: the evidence, why the number isn't higher, and why it isn't lower.

Methodology · How every credence was set

A credence is not a vibe — it's set by a repeatable procedure: start from the strength of the best evidence, then subtract explicit discounts. Two opinions with the same number should be wrong equally often.

  1. Rung 4 — strongestCausal intervention: we changed one thing and the metric moved (e.g. the grep-cap patch). Starts ~90.
  2. Rung 3Direct trace attribution: the cost is read line-by-line out of the artifact (e.g. 35.7K KG read at turn 1). Starts ~85.
  3. Rung 2Structural argument: checkable reasoning about the system (e.g. "no WB case requires semantic search" — verifiable by reading the 5 tasks). Starts ~75.
  4. Rung 1 — weakestExternal analogy: someone else's benchmark on someone else's corpus (PwC, GrepAI, Manus). Corroborates; never carries a number alone. Starts ~60.

Then the discounts, each applied explicitly: small-n (every cell here is n=1–3; ±30% token / ±1-point swings are documented — subtract 5–15), unmeasured confounder (e.g. the codex clip layer — subtract 5–10), agent-compliance uncertainty (codex may ignore instructions — subtract 5–10), and the motivated-reasoning discount: where being right flatters our product or our past work (marked identity-fused), the number is pushed down on purpose, because that's exactly where self-deception lives.

O1
Delivery form — what each search call puts into context — is the #1 token lever. Not retrieval quality. Not the KG.
credence ~85%

The claim, unpacked: of everything semfs can change — embedder, reranker, KG, hints, output style — the payload each semfs grep call injects into context (its size, which bytes, and at what turn) moves the token bill the most.

Why we hold it (Rung 4: causal intervention)We patched exactly one thing — a per-hit render cap — and case 289 fell from 111–145K to 76.8K, the first sub-plain run in the project's history. Before the patch, a single broad grep was returning 142,733 characters, and the documented cap knobs were provably inert (capped 372K chars ≈ uncapped 316K). The field independently converged on the same lever (Firetiger artifacts, Anthropic code-exec −98.7%).
Why not higher than 85Two open holes. (1) The codex clip layer (Exhibit A) is unmeasured — if the harness already truncates aggressively, part of our "savings" was being captured by blind clipping anyway, and the cap's marginal value shrinks. (2) Run 2 of the same config made 16 calls instead of 5 and lost the win — behavior variance can swamp payload effects, which is O2's territory.
Why not lowerThe intervention is the strongest evidence type we have anywhere in this case file, and it replicated directionally at two cap settings (6KB: tokens fall, accuracy holds; 3KB: tokens fall below plain, −1 point). Dismissing it would require explaining away a direct A/B.
Setting the number: Rung 4 start ≈ 90 − 5 (clip confounder) ≈ 85. No identity discount: delivery being the lever neither flatters nor indicts any prior investment.
Killed ifE8 (n=3, hint fixed) shows capped vs uncapped delivery within ±10% on tokens — that would crown the hint (O2) as #1 instead.

credence ~80%

The claim, unpacked: the injected AGENTS.md note — roughly 2KB of prose — determines the agent's turn count, read order, and crawl behavior, and those three things are the cost function. A better hint buys more than a better embedder.

Why we hold it (Rung 3: trace attribution + one intervention)The causal chain is verified link by link in the traces: hint says "read kg/ FIRST" → agent burns 35.7K tokens at turn 1 → greps scoped to kg/ and gets 138 useless chars → confidence collapses → 22.4K os.walk crawl. And we have a documented counterexample in this very project: Claude Code ignored the semfs grep hint entirely. Hint-obedience is agent-specific. Plus the standing identity discount — "the text file matters more than my pipeline" is also a flattering story in reverse (it makes the fix look easy), so it gets the same skepticism.
Why not lowerThe 58K cascade isn't a correlation — each step is a specific line in a specific trace, and the lean-hint intervention moved the total exactly the way the chain predicts.
Setting the number: Rung 3 start ≈ 85 − 5 (compliance uncertainty, n=1–2 on the intervention) ≈ 80. Identity-fused flag noted; the discount is folded into the compliance subtraction.
Killed ifE7's hint ladder (n=3) shows <10% token spread between hint variants on the same seed and binary. Pre-commit: effort moves to silent affordances (E13) instead.

credence ~60% — deliberately modest

The claim, unpacked: combine everything proven so far — clean infra (H1) + visible tree (H2) + capped grep (H4) + lean hint (O2) + suppressed KG (H5/O7) — and the stack wins tokens at accuracy parity on at least 3 of the 5 cases. How we save turn cost — the playbook, written down

Every turn re-pays the whole context, so each mechanism below is worth (scaffold + history) × turns-avoided. The first five are shipped; the sixth is the frontier.

1 · Never command a read (deterministic, shipped)The old hint ordered a 35.7K KG read at turn 1 plus the crawl it triggered (~58K/run). The new hint commands nothing — zero KG reads, zero full-tree crawls in all six scout-class runs.
2 · Make turn 1 terminal-grade (shipped)One semantic grep whose render carries the answer rows inline, capped inside the harness clip window so the model sees all of it. When codex trusts it, the whole task is search → write: 2 calls, 21K tokens.
3 · Keep the escape hatch open but unnecessary (shipped)SEARCH_ONLY=off keeps the real tree readable. It prevents the catastrophic spiral class entirely (the old 45-min timeouts, 23-call flails) — worth more than it costs.
4 · Close known dead-ends early (shipped)The provenance check makes the agent look at task-named sources once, see the 403 stub, report it, and STOP hunting for data that does not exist — w2 burned 6 extra grep variants on exactly that hunt.
5 · Pay bytes once and honestly (shipped)Capped renders + truthful COMPLETE/TRUNCATED markers mean the model never receives a silently gutted blob — gutted results breed re-query turns.
6 · The frontier: the stop signal (E9, next)The residual 2-vs-9-call coin flip is codex's own verify instinct. A static hint can't pin it; per-query confidence in the render ("rank-1 hit, high agreement — no further search needed") is the closed-loop fix. Plus: move the provenance paragraph above the search instructions (compliance was ~50% with it last).
