02 · the lever · eli5 → expert

Why one prompt changed everything

The agent already had the answer.

ELI5  semfs grep handed the agent the exact source records — IDs, dates, numbers — right in its context. It just… wrote a generic summary instead of copying them.

So it kept searching for files it already had.

BASIC  Not trusting that the search was the content, the agent re-ran searches and hunted for the underlying files — burning tokens, never transcribing.

The prompt says: STOP, and copy the values verbatim.

INTER  “The grep results ARE the source content. After a few searches you have what you need — STOP. TRANSCRIBE the exact IDs/dates/numbers verbatim; do not summarize.”

Two effects at once — behavior, not retrieval.

ADV  (a) the re-search loop halts → tokens collapse 5×; (b) transcription replaces summary → accuracy ~doubles. Retrieval was never the problem; behavior was.

Proven by ablation, not asserted.

EXPERT  Same delivery config without the prompt (exp_0003) → 0.24 accuracy at 878K tokens (per-cell up to 2.0M). Prompt alone (exp_0007) → already beats plain. The prompt is the single load-bearing variable.

06 · honest accounting

Caveats & what’s next

📉 Noisy benchmark

n=3; per-rep accuracy swings 0→0.91. Deltas between semfs configs (0.29–0.50) are largely noise — the robust signal is the large margin over plain, confirmed by two independent configs.

🧪 Generalization unvalidated

Held-out cases 95/386/175 are beyond glm-5.1 for both arms (plain 0% at 1–2.3M tokens) → inconclusive, not a semfs loss. Win is confirmed on 53/171 only.

⏱️ compress → timeout risk

SEMFS_GREP_COMPRESS makes per-grep OpenRouter calls → latency → timeouts on grep-heavy cases. The simpler prompt-only config avoids it.

🎯 Real next lever

The prompt is a soft stop — case 95 over-explored 134 calls anyway. A hard search/turn cap (Rust/harness) is the genuine follow-up. The RCA’s FUSE-enum / delivery-form levers are low-EV here (behavior, not delivery, is the bottleneck).

✅ Recommendation

Ship knobs/prompt_only.json (robust, beats plain on both axes) — or knobs/best_exp0002.json for max accuracy on fast cases. The transcription/stop prompt is the essential ingredient; fold it into the semfs grep affordance / MCP-tool hint rather than a per-task prompt.
